# Results

Tabular evaluation outputs generated by the notebook pipeline. These CSV files support the analysis in [`../docs/RESULTS.md`](../docs/RESULTS.md).

---

## File Index

| File | Description |
|------|-------------|
| `model_leaderboard.csv` | All baseline models ranked by cross-domain performance |
| `per_sentiment_performance.csv` | Precision, recall, F1 per sentiment class × platform |
| `compression_by_sentiment.csv` | Reddit vs Twitter linguistic feature means and diffs |
| `calibration_summary.csv` | Expected Calibration Error (ECE) per sentiment class on Twitter |
| `coral_results.csv` | Baseline Logistic Regression vs CORAL-adapted LR |
| `clinical_risk_summary.csv` | F1 drop and missed-rate framing per sentiment class |

> Some local copies may have a ` (1)` suffix from manual re-exports (e.g., `model_leaderboard (1).csv`). The notebook writes canonical names without the suffix on a fresh run.

---

## Schema Reference

### `model_leaderboard.csv`

| Column | Description |
|--------|-------------|
| `Model` | Classifier name |
| `Reddit_Val_Acc` | Accuracy on Reddit validation set |
| `Reddit_Val_F1` | Weighted F1 on Reddit validation |
| `Twitter_Test_Acc` | Accuracy on Twitter test set |
| `Twitter_Test_F1` | Weighted F1 on Twitter test (primary ranking metric) |
| `Acc_Drop` | Reddit val accuracy − Twitter test accuracy |
| `Drop_%` | Relative accuracy drop (%) |
| `Train_Time` | Training time in seconds |

### `per_sentiment_performance.csv`

| Column | Description |
|--------|-------------|
| `sentiment` | Label: -1 (negative), 0 (neutral), or 1 (positive) |
| `platform` | `Reddit` or `Twitter` |
| `precision` | Class precision |
| `recall` | Class recall |
| `f1-score` | Class F1 |

### `compression_by_sentiment.csv`

Per-class rows with Reddit means (`reddit_*`), Twitter means (`twitter_*`), and differences (`diff_*`) for 10 linguistic features. Key columns:

| Column | Description |
|--------|-------------|
| `sentiment` | Label: -1, 0, or 1 |
| `reddit_type_token_ratio` | Mean TTR on Reddit for this class |
| `twitter_type_token_ratio` | Mean TTR on Twitter for this class |
| `diff_type_token_ratio` | Reddit − Twitter TTR difference |
| *(similar columns for 9 other features)* | See [`../data/README.md`](../data/README.md) |

### `calibration_summary.csv`

| Column | Description |
|--------|-------------|
| `sentiment` | Label: -1, 0, or 1 |
| `ECE` | Expected Calibration Error on Twitter test |

### `coral_results.csv`

| Column | Description |
|--------|-------------|
| `Model` | `Baseline LR` or `CORAL LR` |
| `Acc` | Twitter test accuracy |
| `F1` | Twitter test weighted F1 |

### `clinical_risk_summary.csv`

| Column | Description |
|--------|-------------|
| `Sentiment_Label` | -1, 0, or 1 |
| `Sentiment_Name` | Negative / Neutral / Positive |
| `Reddit_F1` | F1 on Reddit validation |
| `Twitter_F1` | F1 on Twitter test |
| `F1_Drop` | Reddit F1 − Twitter F1 |
| `Missed_Rate` | Fraction of class samples misclassified on Twitter |
| `Clinical` | Whether class is flagged as high-risk (Negative = True) |

## Regenerating Results

Run the full notebook:

```bash
jupyter notebook notebooks/ML_Project_final.ipynb
```

Or on Kaggle: [`../notebooks/ML_Project_kaggle.ipynb`](../notebooks/ML_Project_kaggle.ipynb)

Results are written to `{BASE}/results/` where `BASE` is:

| Environment | Path |
|-------------|------|
| Colab | `/content/sentiment_project/results/` |
| Kaggle | `/kaggle/working/sentiment_project/results/` |
| Local | Configured in notebook path setup |
